## QA oprogramowania (studenckiego) * start projektu, wymagania, architektura * implementacja * QA * `if` nie ma jakości `then` ewakuacja * oddanie Nikogo innego nie stać na wywalenie ~skończonego projektu
## O jakość trzeba dbać przez cały czas trwania projektu
## Jak?
### Coś co znacie - unit testing Zadanie domowe: Jeśli nie znasz, poznaj definicje - Unit of work (w kontekście unit testów) - TDD - Test First
- Unit !== plik - Test First to kontrowersyjny temat - Dobre testowanie jednostkowe jest trudniejsze niż myślisz
### Jak nie zabetonować projektu - nie testuj szczegółów implementacji tylko funkcjonalność > Funkcja sort ma poprawnie sortować, a nie przestawiać elementy po kolei według określonego w teście algorytmu. - nie testuj czy kod wywołuje metodę, ale czy korzysta z jej efektów - rób asercje tylko na prawdziwe rezultaty, nie stan całego świata

## Pisanie testowalnego kodu - O tym jest TDD - mądra enkapsulacja - Co z tego, że działa skoro przed deployem nie udowodnisz? >Test First przy naprawianiu bugów, ale mało realne przy budowaniu projektu od zera >(wspominałem o betonowaniu?)
### Kilka definicji - Mock - Stub - Fixture
### Myśleliście że to wszystko? - Happy path - Negative test - Coverage - Branch coverage
## Typy testów według "jak" - unit test - integration test - functional test - systemic / end-to-end - stress test > Różne zastosowania, różny koszt utrzymania
### Typy testów według "dlaczego" - Regression test - Acceptance test - Exploratory test - Smoke test
### Continous Integration Jeśli nie chcesz używać CI, zmień branżę już teraz.
### Automatyzacja - API - UI (Selenium/OS) - Ogórki [Given/When/Then](https://github.com/cucumber/cucumber/wiki/Gherkin)
``` Feature: Serve coffee Coffee should not be served until paid for Coffee should not be served until the button has been pressed If there is no coffee left then money should be refunded Scenario: Buy last coffee Given there are 1 coffees left in the machine And I have deposited 1$ When I press the coffee button Then I should be served a coffee ```
